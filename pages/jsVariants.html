JavaScript Variables | Var - Const - Let

In JavaScript, var, let, and const are used to declare variables, but they have key differences in scope, reassignment, and hoisting.

    var (Function-scoped, Can be Reassigned)

  • Function-scoped (available inside the function where it is declared)
  • Can be redeclared and reassigned
  • Hoisted (moved to the top during execution but initialized as undefined)
  • Can cause unexpected behavior due to lack of block scope

    • let (Block-scoped, Can be Reassigned)

  • Block-scoped (only available inside {} where it is declared)
  • Can be reassigned but cannot be redeclared in the same scope
  • Hoisted but not initialized (causes a ReferenceError if accessed before declaration)

    • const (Block-scoped, Cannot be Reassigned)

  • Block-scoped (like let)
  • Cannot be reassigned (but objects and arrays can be modified)
  • Must be initialized during declaration
